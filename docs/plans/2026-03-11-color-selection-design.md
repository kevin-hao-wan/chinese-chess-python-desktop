# AI 执棋方选择功能设计文档

**日期**: 2026-03-11
**主题**: color-selection
**状态**: 已批准

---

## 概述

为中国象棋游戏增加执棋方选择功能，允许玩家选择执红先行（默认）或执黑后行（AI 先手）。

---

## 目标

- 在游戏开始前提供执棋方选择功能
- 支持人执红先行（默认）和人执黑后行两种模式
- 选择人执黑后行时，AI 自动走第一步
- 选择应在游戏开始前进行，游戏结束后可重新选择

## 非目标

- 不改变现有棋力算法
- 不支持游戏中途更换执棋方
- 不涉及网络对战功能

## 关键组件

### 1. GameBridge 类 (src/gui/bridge.py)

**新增属性:**
- `player_color`: 记录玩家选择的颜色（Color.RED 或 Color.BLACK）

**修改方法:**
- `new_game(player_color)`: 接受玩家颜色参数，根据选择初始化游戏
- `onCellClicked()`: 根据玩家颜色判断可操作方，而非硬编码红方
- `_get_turn_text()`: 动态显示回合信息，支持"AI 思考中"提示
- `_ai_first_move()`: AI 先手时自动走棋

### 2. 主界面 (src/gui/main.qml)

**修改:**
- "新游戏"按钮点击时弹出选择对话框
- 接收玩家选择后调用 `gameBridge.startNewGame(playerColor)`

**新增:**
- 处理选择对话框的确认信号

### 3. 选择对话框 (src/gui/ColorSelectionDialog.qml)

**组件:**
- 两个 RadioButton：执红先行、执黑后行
- 确认/取消按钮
- 使用 QSettings 记住上次选择

## 数据流

```
用户点击"新游戏"
    ↓
弹出 ColorSelectionDialog
    ↓
用户选择 → 点击确认
    ↓
gameBridge.startNewGame(playerColor)
    ↓
初始化棋盘，设置 current_turn
    ↓
如果 playerColor == BLACK:
    显示"红方思考中"
    触发 AI 走棋
    ↓
轮到玩家操作
```

## 关键代码变更

### 状态管理

```python
# 新增属性
self._player_color = Color.RED  # 默认为红方

# 修改回合判断
if self._board.current_turn == self._player_color:
    # 玩家回合
else:
    # AI 回合
```

### 走棋流程

1. 玩家走棋后检查游戏结束，然后触发 AI
2. AI 走棋后检查游戏结束
3. 如果 AI 先手，初始化后立即触发 `_ai_move()`

### UI 提示

- `currentTurn` 根据当前回合和玩家颜色动态显示
- "红方思考中"在 AI 计算时显示

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| AI 先手时用户等待体验不佳 | 添加"AI 思考中"提示 |
| 选择对话框增加操作步骤 | 记住上次选择，减少重复操作 |
| 测试用例需要覆盖两种模式 | 为两种执棋方模式分别编写测试 |

## 决策记录

**决策 1：选择时机**
- **选择**: 在游戏开始前（主菜单或新游戏对话框）进行执棋方选择
- **理由**: 避免游戏进行中的复杂状态管理，符合用户心理模型

**决策 2：UI 实现方式**
- **选择**: 在主窗口添加一个"新游戏"按钮，点击后弹出选择对话框
- **理由**: 清晰直观，不会干扰正在进行的对局

**决策 3：AI 先手触发机制**
- **选择**: 初始化游戏时根据玩家选择设置当前回合，如果是 AI 先手则立即触发 AI 走棋
- **理由**: 保持现有回合管理逻辑，仅需在初始化时增加条件判断
